- `agent create my-agent --mode package` creates a reusable Node agent module.
- `agent create my-agent-service --mode service` creates a Next.js chat service with the same agent module inside it.

## Extension Points

- Add tools in `src/tools/` and export them from `src/tools/index.ts`.
- Configure `OPENAI_API_KEY`, `LLM_BASE_URL`, and `LLM_MODEL` in your environment. `LLM_BASE_URL` defaults to `https://api.openai.com/v1`; `LLM_MODEL` defaults to `agent.config.ts`.
- Use `createAgent({ modelProvider })` to replace the LLM provider.
- Use `createAgent({ memoryStore })` to replace the default in-memory session store.
- Extend guard behavior in `src/agent/guard.ts`.
- Extend planning in `src/agent/planner.ts`.

## Node Integration

```ts
import { createAgent } from './src/index.js';

const agent = createAgent();
const result = await agent.run({ input: 'calculate 1 + 2', sessionId: 's1' });
console.log(result);
```

Custom tool:

```ts
import { z } from 'zod';
import { createAgent, type ToolDefinition } from './src/index.js';

const echoTool: ToolDefinition = {
  name: 'echo.say',
  description: 'Echo text.',
  permission: 'public',
  timeoutMs: 1000,
  retry: 0,
  inputSchema: z.object({ text: z.string() }),
  outputSchema: z.object({ text: z.string() }),
  async handler(input) {
    return input;
  },
};

const agent = createAgent({ tools: [echoTool] });
```

Trace files are written to `.agent-traces/<traceId>.json`.

## Service UI

- `/` opens the Agent chat UI.
- `/traces` opens the local trace viewer.
- `POST /api/agent` runs the Agent.
- `GET /api/traces` lists trace summaries.
- `GET /api/traces/[traceId]` reads a trace record.

The default model provider is OpenAI-compatible and requires `OPENAI_API_KEY`. The generated Agent does not include an LLM mock or fallback response.
